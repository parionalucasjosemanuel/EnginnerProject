Programación Competitiva en C++

Clase 1: Complejidad Algorítmica y STL

📘 TEORÍA COMPLETA

La complejidad algorítmica define cuánto tiempo tarda un algoritmo en ejecutarse dependiendo del tamaño de entrada. En programación competitiva esto es CRÍTICO porque los jueces tienen límites estrictos de tiempo (1–3 segundos).

Conceptos fundamentales:

  • O(1) → constante (acceso directo)
  • O(log n) → búsqueda binaria
  • O(n) → recorrido simple
  • O(n log n) → algoritmos eficientes de ordenamiento
  • O(n²) → doble bucle (limitado)

La STL (Standard Template Library) es clave:

  • vector → arrays dinámicos
  • map → hashing + orden
  • set → valores únicos ordenados
  • priority_queue → heap

🧠 EJEMPLO REAL

vector v = {5,2,8};
sort(v.begin(), v.end());

🧩 PROBLEMAS

  • Two Sum optimizado
  • Búsqueda binaria
  • Frecuencias con map

Clase 2: Grafos (BFS, DFS, Dijkstra)

📘 TEORÍA

Un grafo es una estructura que modela relaciones entre nodos. Se usa en redes, rutas, juegos, etc.

Algoritmos principales:

  • DFS: exploración profunda (recursivo)
  • BFS: exploración por niveles
  • Dijkstra: camino mínimo

🧠 COMPLEJIDAD

  • BFS/DFS → O(V + E)
  • Dijkstra → O((V+E) log V)

🧩 PROBLEMAS

  • Labyrinth
  • Shortest Path
  • Connected Components

Programación Competitiva en Java

Clase 1: Entrada/Salida Rápida y Estructuras

📘 TEORÍA COMPLETA

En Java, la velocidad de entrada/salida es crítica. Usar Scanner puede causar TLE. Se recomienda:

  • BufferedReader para lectura rápida
  • PrintWriter para salida eficiente
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
String[] input = br.readLine().split(" ");

Estructuras importantes:

  • ArrayList → listas dinámicas
  • HashMap → búsqueda O(1)
  • TreeMap → ordenado
  • PriorityQueue → heaps

🧩 PROBLEMAS

  • Conteo de frecuencias
  • Ordenamientos personalizados
